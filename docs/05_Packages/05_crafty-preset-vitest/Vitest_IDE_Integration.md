Vitest's IDE integration relies in general on the `vitest.config.*` files.

With the `crafty ide` command, a configuration file is generated to be read by
your IDE.

> `vitest.config.crafty.mjs` is generated by Crafty, and existing
> `.vscode/settings.json` files are merged instead of replaced.

`crafty-preset-vitest` generates `vitest.config.crafty.mjs` and
`.vscode/settings.json`.

If a native `vitest.config.*` file already exists,
`vitest.config.crafty.mjs` is written next to it and loads it before merging
the Crafty-generated configuration.

If `.vscode/settings.json` already exists, Crafty merges the
`vitest.rootConfig` setting into it instead of replacing it.

## IDE plugins

- Visual Studio Code: Use a Vitest extension that reads `vitest.rootConfig`
