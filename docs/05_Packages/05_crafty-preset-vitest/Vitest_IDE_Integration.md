Vitest's IDE integration relies in general on the `vitest.config.*` files.

With the `crafty ide` command, Crafty writes the IDE-facing configuration file
that your editor should use.

> `vitest.config.mjs` is generated by Crafty, alternative Vitest config
> filenames are removed, and existing `.vscode/settings.json` files are merged
> instead of replaced.

`crafty-preset-vitest` generates `vitest.config.mjs` and
`.vscode/settings.json`.

`crafty test` ignores official `vitest.config.*` files. When you run
`crafty ide`, Crafty takes ownership of the IDE-facing config by writing
`vitest.config.mjs` and removing alternative filenames such as
`vitest.config.js`, `vitest.config.cjs`, `vitest.config.mts`,
`vitest.config.ts`, `vitest.config.cts`, and the legacy
`vitest.config.crafty.mjs`.

If `.vscode/settings.json` already exists, Crafty merges the
`vitest.rootConfig` setting into it instead of replacing it.

## IDE plugins

- Visual Studio Code: Use a Vitest extension that reads `vitest.rootConfig`
