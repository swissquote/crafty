Vitest's IDE integration relies in general on the `vitest.config.*` files.

With the `crafty ide` command, Crafty writes the IDE-facing configuration file
that your editor should use.

> `vitest.config.mjs` is generated by Crafty and alternative Vitest config
> filenames are removed.

`crafty-preset-vitest` generates `vitest.config.mjs`.

`crafty test` ignores official `vitest.config.*` files. When you run
`crafty ide`, Crafty takes ownership of the IDE-facing config by writing
`vitest.config.mjs` and removing alternative filenames such as
`vitest.config.js`, `vitest.config.cjs`, `vitest.config.mts`,
`vitest.config.ts`, and `vitest.config.cts`.

Crafty does not write or merge `.vscode/settings.json` for Vitest.

If your editor extension requires a `vitest.rootConfig` setting, configure it
manually and point it to `vitest.config.mjs`.

## IDE plugins

- Visual Studio Code: Use a Vitest extension that can discover
  `vitest.config.mjs`, or configure `vitest.rootConfig` manually when your
  extension requires it.
